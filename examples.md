# 📚 LRL — Examples

This document provides practical examples of how the Legacy Release License (LRL) applies to real-world scenarios.

---

## 🎮 Example 1 — Abandoned Game (Multiplayer Revival)

**Project:** Online Multiplayer Game
**Phase I:** 2015–2020
**Phase II:** 2020–2035
**Phase III:** 2035+

### Scenario

The developer shuts down official servers in 2020 and stops releasing updates.

### Under LRL

* Players can:

  * Reverse engineer networking protocols
  * Create private servers
  * Maintain gameplay through community patches

* Restrictions:

  * No monetization (no paid servers, no selling mods)
  * No use of official branding as if it were official

### Outcome

The game remains playable and evolves through community support instead of disappearing.

## 🔫 Example 2 — Mod Ecosystem (Inspired by Counter-Strike)

**Project:** FPS Game Engine
**Phase I:** Active
**Phase II:** Triggered after abandonment

### Scenario

The original game is no longer supported, but has a strong modding community.

### Under LRL

* Developers can:

  * Create total conversions
  * Build new game modes
  * Extend engine capabilities

* Restrictions:

  * Mods must remain non-commercial during Phase II
  * Must include attribution

### Outcome

The ecosystem evolves similarly to how mods like Counter-Strike originally emerged — but now legally protected after abandonment.

## 🧟 Example 3 — Post-Support Game Maintenance (Inspired by Left 4 Dead)

**Project:** Cooperative Survival Game
**Phase I:** 2008–2015
**Phase II:** 2015–2030
**Phase III:** 2030+

### Scenario

Official updates stop, but the player base remains active.

### Under LRL

* Community can:

  * Fix bugs
  * Rebalance gameplay
  * Maintain compatibility with modern systems

* Restrictions:

  * No selling modified versions
  * No claiming official status

### Outcome

The game continues receiving updates long after official support ends, without legal uncertainty.

**Phase III:** 2033+

### Scenario

The company shuts down and the software is no longer maintained.

### Under LRL

## 🧩 Example 5 — Game Engine Revival

**Project:** Proprietary Game Engine
**Phase I:** Active development
**Phase II:** After abandonment
**Phase III:** Full release

### Scenario

The engine is discontinued and replaced by a new one.

### Under LRL

* Phase II:

  * Community maintains compatibility
  * Tools and extensions are created

* Phase III:

  * Source code becomes available
  * Engine can be studied and reused freely

### Outcome

The engine becomes part of technical history instead of being lost.

---

## 🧠 Key Takeaway

The LRL does not interfere with active development or commercial viability.

Instead, it ensures that once a project is abandoned:

* It does not disappear
* It does not become legally untouchable
* It can evolve beyond its original lifecycle

---

## In one sentence

LRL allows projects to live twice:

* First as a product
* Then as a legacy
